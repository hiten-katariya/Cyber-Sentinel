FEATURE #3: QUARANTINE FEATURE
================================================================================

WHAT IT IS:
Automatically move files containing sensitive data to a secure quarantine folder.

HOW TO USE:

ENABLE QUARANTINE:
1. Go to Dashboard or Advanced tab
2. Click "Toggle Quarantine" or "Enable Quarantine"
3. Status shows "ENABLED" in green

WHAT HAPPENS:
- Files with sensitive data are moved to quarantine/ folder
- Original location is preserved in metadata
- Timestamped folders keep quarantine organized
- Metadata file (.json) records details

QUARANTINE STRUCTURE:
  quarantine/
    20251222_153045/
      sensitive_file.txt
      sensitive_file.txt.metadata.json

RESTORE FILES:
Manually move files back from quarantine folder to original location.

BENEFITS:
- Prevent accidental data exposure
- Isolate sensitive files immediately
- Maintain audit trail
- Reversible action

FEATURE #4: WHITELIST MANAGEMENT (False Positives)
================================================================================

WHAT IT IS:
Skip scanning for specific files or patterns to reduce false positives.

HOW TO USE:

ADD TO WHITELIST:
1. Go to Advanced tab
2. Enter file path or pattern in text box
3. Click "Add to Whitelist"

EXAMPLES:
- Exact file: C:\Users\John\Documents\safe_file.txt
- Pattern: *.log (skip all log files)
- Pattern: C:\SafeFolder\* (skip entire folder)

REMOVE FROM WHITELIST:
1. Find item in whitelist
2. Click "Remove" button

VIA API:
  # Add to whitelist
  curl -H "X-API-Key: KEY" -X POST http://127.0.0.1:5000/api/whitelist/add \
    -H "Content-Type: application/json" -d '{"item": "*.log"}'
  
  # Remove from whitelist
  curl -H "X-API-Key: KEY" -X POST http://127.0.0.1:5000/api/whitelist/remove \
    -H "Content-Type: application/json" -d '{"item": "*.log"}'

BENEFITS:
- Reduce alert fatigue
- Skip known safe files
- Speed up scanning

FEATURE #5: BACKUP & RECOVERY
================================================================================

WHAT IT IS:
Automatic backup of configuration, whitelist, and statistics.

HOW TO USE:

CREATE BACKUP:
1. Go to Advanced tab
2. Click "Create Backup Now"
3. File is saved to backups/ folder

BACKUP CONTENTS:
- Configuration (monitored paths, destinations, email)
- Whitelist entries
- Statistics summary
- Timestamp

RESTORE FROM BACKUP:
1. Go to Advanced tab
2. Enter backup file path
   Example: backups/dlp_config_backup_20251222_153045.json
3. Click "Restore Backup"
4. Confirm the action

VIA API:
  # Create backup
  curl -H "X-API-Key: KEY" -X POST http://127.0.0.1:5000/api/backup/create
  
  # Restore backup
  curl -H "X-API-Key: KEY" -X POST http://127.0.0.1:5000/api/backup/restore \
    -H "Content-Type: application/json" \
    -d '{"backup_file": "backups/dlp_config_backup_20251222_153045.json"}'

BENEFITS:
- Disaster recovery
- Configuration versioning
- Easy rollback
- Test changes safely

API REFERENCE - NEW ENDPOINTS
================================================================================

STATISTICS:
  GET  /api/statistics               Get alert statistics

EXPORT:
  GET  /api/export/csv               Export alerts to CSV
  GET  /api/export/json              Export alerts to JSON

WHITELIST:
  GET  /api/whitelist                Get whitelist items
  POST /api/whitelist/add            Add item to whitelist
  POST /api/whitelist/remove         Remove item from whitelist

QUARANTINE:
  GET  /api/quarantine/status        Check if quarantine is enabled
  POST /api/quarantine/enable        Enable quarantine
  POST /api/quarantine/disable       Disable quarantine

BACKUP:
  POST /api/backup/create            Create configuration backup
  POST /api/backup/restore           Restore from backup

All endpoints require X-API-Key header except /api/status.

BEST PRACTICES

1. CREATE REGULAR BACKUPS

   - Test restore process

2. USE WHITELIST WISELY

3. MONITOR QUARANTINE
   - Review quarantined files regularly
   - Restore false positives
   - Delete confirmed sensitive data

4. EXPORT REPORTS
   - Export monthly for compliance
   - Share with security team
   - Keep historical records

5. REVIEW DASHBOARD
   - Check daily for anomalies
   - Track trends over time
   - Adjust policies based on data


================================================================================
TROUBLESHOOTING
================================================================================

"Quarantine not working":
→ Check that quarantine is enabled (Dashboard or Advanced tab)
→ Verify write permissions on quarantine folder
→ Check logs for quarantine errors

"Whitelist not applying":
→ Ensure pattern syntax is correct (* for wildcards)
→ Restart monitoring after adding to whitelist
→ Check logs to see if files are marked as whitelisted

"Backup restore failed":
→ Verify backup file path is correct
→ Check backup file is valid JSON
→ Ensure you have write permissions

"Export files not found":
→ Check project root folder
→ Look for files with timestamp: alerts_export_*.csv
→ Check logs for export errors

SECURITY CONSIDERATIONS
================================================================================

QUARANTINE FOLDER:
- Restrict access permissions
- Monitor for unauthorized access
- Regular cleanup of old quarantined files

BACKUP FILES:
- Contain sensitive configuration
- Store securely (chmod 600)
- Encrypt if storing remotely

EXPORTS:
- May contain sensitive alert details
- Don't share without redaction
- Delete old exports regularly
