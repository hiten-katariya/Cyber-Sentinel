CYBER SENTINEL - DEMONSTRATION SCRIPT FOR PRESENTATION
======================================================

PREPARATION (Before presentation)
---------------------------------
1. Start Cyber Sentinel: python dlp_web.py
2. Open browser to: http://127.0.0.1:5000
3. Have the monitored_data/ folder open in File Explorer (for visual reference)
4. Clear any previous alerts: Advanced page → Clear All Alerts
5. Set theme to your preference (Dark/Light toggle in header)

DEMONSTRATION FLOW (15-20 minutes)
===================================

PART 1: INTRODUCTION & OVERVIEW (2 minutes)
-------------------------------------------
"Today I'm presenting Cyber Sentinel, an advanced Data Loss Prevention system 
that protects sensitive data by monitoring files and detecting security threats 
in real-time."

Show: Dashboard page
- Point out the clean, modern interface
- Explain the statistics cards showing system metrics
- Mention the dark/light theme toggle (demonstrate it)

PART 2: SYSTEM CONFIGURATION (3 minutes)
-----------------------------------------
Go to: Configuration page

"First, let's configure what folders we want to monitor and set up alerts."

1. Show the monitored paths:
   - Point to: monitored_data folder (already configured)
   
2. Show authorized destinations:
   - Explain: "These are safe external locations where files can be transferred"
   - Example: internal_server.com, backup.company.com

3. Email alerts configuration:
   - Show SMTP settings
   - Explain: "System sends email notifications when threats are detected"

4. Show the folder picker:
   - Click "Browse Folder" button
   - Demonstrate how users can easily add folders to monitor

"The system uses regex patterns to detect 8 types of sensitive data:
- Credit card numbers
- Social Security Numbers (SSN)
- Email addresses
- Phone numbers
- IP addresses
- API keys
- and more"

PART 3: REAL-TIME MONITORING DEMO (5 minutes)
----------------------------------------------
Go to: Monitoring page

"Now let's see Cyber Sentinel in action with real-time monitoring."

1. Show the current status (STOPPED)

2. Click "Start Monitoring"
   - Status pill changes to green "MONITORING"
   - Toast notification appears: "Monitoring started"
   - Show the keyboard shortcut hint: "You can also use Ctrl+S to start"

3. Wait 5-10 seconds for automatic scan
   - Point out: "The system automatically scans at regular intervals"

4. Click "Manual Scan" button
   - Progress bar appears
   - Shows scan duration when complete
   - Toast notification: "Scan completed in X seconds"

5. Demonstrate drag-and-drop testing:
   - Open File Explorer with monitored_data/
   - Drag customer_database.txt to the drop zone
   - Show how file path auto-fills

6. Test Transfer feature:
   - File: monitored_data/customer_database.txt
   - Destination: unknown_external_server.com
   - Click "Test Transfer"
   - Result: "Transfer BLOCKED" (shown in red)
   
   Then test with authorized destination:
   - Destination: internal_server.com
   - Result: "Transfer ALLOWED" (shown in green)

7. Show quick actions:
   - Click "Quick Scan Downloads" (if you have files there)
   - Show the instant feedback and progress

8. Click "Stop Monitoring"
   - Status changes back to STOPPED
   - Show keyboard shortcut: Ctrl+X

5. Demonstrate print view:

TROUBLESHOOTING TIPS
====================
If no alerts appear:
- Manually scan again
- Check Configuration → Monitored Paths is set to ./monitored_data
- Verify files exist in monitored_data folder

If theme toggle doesn't work:
- Hard refresh: Ctrl+Shift+R
- Check browser console for errors

If API key prompt appears:
- Check dlp_api_key.txt file in project root
- Copy the key and paste when prompted
- It will be saved in browser localStorage
