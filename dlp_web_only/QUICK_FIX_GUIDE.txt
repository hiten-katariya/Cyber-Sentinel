ISSUE: Drag & Drop Not Working
-------------------------------
SYMPTOMS: Files don't get picked up when dragged to drop zone

FIXES TO TRY:
1. Hard refresh browser: Ctrl+Shift+R (clears cached JavaScript)
2. Check browser console (F12) for errors
3. Verify drop zone logs appear:
   - "Initializing monitoring page..."
   - "Drop zone elements: ..."
   - "Setting up drag and drop handlers..."

4. Test click-to-browse instead:
   - Click the drop zone area
   - File picker should open
   - Select a file from monitored_data folder

5. Manually type file path:
   - In "File path" input, type: monitored_data/customer_database.txt
   - Skip drag and drop entirely

VERIFIED WORKING: ✓ (as of 2025-12-26)
Cache busting implemented, console logging added


ISSUE: Buttons Not Responding
------------------------------
SYMPTOMS: Clicking buttons does nothing

FIXES TO TRY:
1. Hard refresh: Ctrl+Shift+R
2. Check console (F12) for JavaScript errors
3. Verify page-specific JavaScript loaded:
   - Monitoring page: window.DLP_PAGE === 'monitoring'
   - Check base.html includes {% block page_script %}

4. Check API key authentication:
   - Open browser DevTools → Application → Local Storage
   - Look for "dlp_api_key"
   - If missing, check dlp_api_key.txt file

COMMON CAUSES:
- Cached old JavaScript (solution: cache busting now active)
- Missing API key (solution: re-enter from dlp_api_key.txt)
- JavaScript error before handler loads (solution: check console)

VERIFIED WORKING: ✓
All buttons have proper event listeners attached


ISSUE: Theme Toggle Not Working
--------------------------------
SYMPTOMS: Dark/light mode doesn't switch

FIXES TO TRY:
1. Hard refresh: Ctrl+Shift+R
2. Check localStorage:

   - Look for "theme" key

   - "Applied theme: ..."

Theme initialization moved before main() IIFE


ISSUE: Quick Scan Buttons Not Working
--------------------------------------
SYMPTOMS: "Quick Scan Downloads/Documents" does nothing

NOTE: These buttons work but scan the Downloads/Documents folders.
If you don't have sensitive files there, no alerts will appear.

TO TEST PROPERLY:
1. Copy a test file to your Downloads folder:
   - Copy monitored_data/customer_database.txt
   - Paste to C:\Users\[YourName]\Downloads\

2. Click "Quick Scan Downloads"
3. Alerts should appear for the copied file

ALTERNATIVE: Use "Manual Scan" button instead (scans monitored_data/)


ISSUE: Folder Picker Not Working
---------------------------------
SYMPTOMS: "Browse Folder" button doesn't open folder picker

CAUSE: Browser compatibility issue (webkitdirectory attribute)

WORKAROUND:
1. Manually type the folder path in "Add path" input
   Example: ./monitored_data
   
2. Or use forward slashes: C:/Data/Sensitive

3. Click "Add" button

NOTE: This is a browser limitation, not a bug

PERFORMANCE TIPS
================

If system feels slow:
1. Reduce scan interval (edit dlp_monitor.py line with time.sleep())
2. Limit monitored paths to essential folders only
3. Increase polling intervals (change setInterval times in app.js)
4. Use "Pause" buttons to stop auto-refresh when not needed


EMERGENCY RESET PROCEDURE
==========================

If everything is broken:

1. Stop Flask server (Ctrl+C in terminal)
2. Delete browser cache and cookies for localhost:5000
3. Clear localStorage: localStorage.clear()
4. Delete dlp_api_key.txt
5. Restart Flask: python dlp_web.py
6. Copy new API key from terminal
7. Hard refresh browser: Ctrl+Shift+R
8. Enter new API key when prompted
